🎯 **Objective**:

Build a weekly GA4 performance dashboard to monitor acquisition, engagement, retention, and drop‑off trends using SQL‑driven BigQuery transformations.

**🛠 Tools Used:**

Power BI • BigQuery • SQL • GA4

**🔍 Process:**

- Extracted GA4 event + user profile data from BigQuery

- Applied date‑range filters to minimise query cost and improve performance

- Flattened GA4’s nested event structure using SQL for analysis readiness

- Cleaned, standardised, and modelled all data in SQL before loading into Power BI

- Built a structured semantic model with a 5‑week rolling date selector

- Designed a 2‑page dashboard (Overview + Deep Dive) aligned to stakeholder needs

- Enabled full interactivity so each visual dynamically filters the rest

**📈 Key Insights Delivered**

- Clear visibility into total vs new users each week

- Identification of top traffic sources and mediums driving sessions

- Engagement peaks by hour, weekday, and content type

- Week‑0 cohort retention tracked across Weeks 1–4

- High‑exit pages flagged as drop‑off risk zones

**🧾 Impact**
The dashboard provides a single source of truth for user behaviour trends. It enables teams to make informed decisions about content strategy, acquisition channels, and product experience by replacing fragmented reporting with a consistent, automated weekly view.

📦 **Deliverables**

- 2‑page Power BI dashboard

- BigQuery SQL transformation scripts

- Parameterised date‑range slicer

- Weekly refresh setup
